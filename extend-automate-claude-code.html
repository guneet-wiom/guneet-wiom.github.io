Subagents

Smart delegation — why do everything yourself?

Section 2: Subagents. One of the most powerful features in Claude Code. Instead of filling your main conversation, Claude sends a helper with its own context window.
The problem

Context fills up

Claude reads 50 documents to find one answer. All 50 stay in context, diluting attention. (The n-squared problem from Session 1.)

One thing at a time

While Claude investigates, you wait. You cannot give it another task until the first finishes.

Loses track

Long investigations make Claude forget your original question. It finishes researching but lost the thread.

Three problems. Context fills up with research material. You are blocked doing one thing. Long investigations cause Claude to forget your original question. The solution is delegation.
The right tool

Skill vs Subagent

Both are powerful. Different jobs.

Use a Skill when...

You want reusable instructions that run in your main conversation.

"How to write a weekly report"
"How to format a sales proposal"
"How to draft a Slack announcement"

Think: a recipe you follow yourself

Use a Subagent when...

"Not sure which approach works? Let me try both..."
Two subagents test two options. You pick the winner. Zero wasted effort in your chat.

"Compare competitor pricing AND draft proposal? One at a time."
Subagent researches. You keep drafting. Both happen at once.

"Search 30 Slack channels → conversation unusable"
Subagent searches separately. Your conversation stays clean.

Quick test: Will the task flood your conversation with file reads and search results? Subagent. Is it a repeatable workflow you run yourself? Skill.
Critical distinction. Skills are instructions that run in YOUR conversation — like a recipe you follow. Subagents are helpers with their own brain — like sending an intern. Use a skill when you want to DO the task with Claude. Use a subagent when you want Claude to DELEGATE so your main conversation stays clean. Quick test: will it flood your conversation with investigation noise? Subagent. Repeatable workflow? Skill.
The solution

With subagents

Your main Claude sends helpers. Each gets its own brain. Only a summary comes back.

The intern analogy: Like sending 3 interns to research different things simultaneously. Each brings you a one-page summary. You never left your desk.
This is from Anthropic official docs. The Main Agent is your conversation. It spawns subagents — each with their own context window. They do the work separately, produce results, and report back. Your main conversation stays clean. Like sending interns to research — they come back with summaries, you never left your desk. Notice: 3 subagents running in parallel. That is the power — Claude can delegate to multiple helpers at once.

Two ways to call your subagent. Explicit: say "use agent-name to do X" in your prompt. Implicit: if the task matches the agent's description, Claude may pick it automatically. Explicit is more reliable — always name the agent if you want to be sure it runs.
Best practice

Test it immediately

Don't assume it works — verify right after creation

VERIFY THESE 3 THINGS
1. Signature appears in the response
2. Output matches the agent's purpose
3. It didn't fall back to the default agent
COMMON FAILURE
Agent created but never tested.
Weeks later: "Why isn't it working?"
Answer: it never was.
Immediately after creating an agent, run a test prompt. Check three things: the signature appears, the output matches the purpose you defined, and it didn't silently fall back to the default agent. Most broken agents are never tested — people assume creation means it works. It doesn't. Test once, save yourself hours of confusion later.
Best practice

Always add a signature

The only way to confirm your custom agent actually ran

Add this as the first line of your agent's instructions:
Always start your response with: "🤖 [finance-analyst running]"
WITH SIGNATURE
🤖 [finance-analyst running]
Here are the Q4 results...
WITHOUT SIGNATURE
Here are the Q4 results...
(Was this your agent or the default?)
No signature = no proof. Claude doesn't show which agent handled a task. The signature is your only confirmation.
Critical best practice. Claude Code doesn't tell you which agent ran. If you don't see the signature, the built-in general-purpose agent handled it instead of yours. Always add the signature as the first instruction in any custom agent.

When to use what

Subagents vs Agent Teams

Both delegate work. Different levels of coordination.

Subagents

You stay in charge. Each subagent does one task and reports back to you.

Best for:

"Check deployments" — one job, one agent
"Research + draft" — parallel tasks, you combine
"Try two approaches" — you pick the winner

Agent Teams

A team lead agent coordinates multiple agents. They talk to each other, not just to you.

Best for:

"Build this feature end-to-end" — planning + coding + testing
"Audit the whole codebase" — split by area, merge findings
"Research a topic deeply" — multiple angles, synthesized

Rule of thumb: If you want to review each result yourself → subagents. If the task needs agents to coordinate with each other → agent team.
Subagents are like sending interns on separate errands — they report to you. Agent teams are like a project team with a lead — they coordinate amongst themselves. Use subagents for simple parallel tasks where you combine the results. Use agent teams for complex multi-step work where agents need to hand off to each other.
